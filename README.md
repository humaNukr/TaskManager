# KMA TaskManager

## Опис проєкту
Система управління завданнями, побудована за принципами багатошарової архітектури.

## Структура рішення
Проєкт розділений на декілька бібліотек для дотримання принципу **Single Responsibility**:

1. **Common**: Містить переліки (`Enums`), спільні для всіх шарів.
2. **DataModels**: Класи для "сирого" зберігання даних. Не містять обчислювальних полів та логіки відображення.
3. **UIModels**: Моделі для представлення даних у користувацькому інтерфейсі. Містять обчислювані властивості (прогрес, статус прострочення) та логіку форматування.
4. **Services**: Шар бізнес-логіки. Містить:
   - `MockStorage`: Штучне статичне сховище даних.
   - `Mappers`: Логіка перетворення даних між шарами (Data <-> UI).
   - `TaskService` / `ProjectService`: Сервіси для фільтрації та отримання даних.
5. **ConsoleApp**: Клієнтський додаток для взаємодії з користувачем через консоль.

## Ключові принципи
- **Інкапсуляція**: Доступ до штучного сховища `MockStorage` має тільки шар сервісів.
- **Ізоляція рівнів**: Об’єкти `DataModel` першого рівня не містять колекцій об’єктів другого рівня, зв’язок реалізовано через ідентифікатори (Guid).
- **Lazy Loading**: Завдання завантажуються лише після вибору конкретного проєкту.
